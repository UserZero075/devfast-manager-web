Documentación

Guía práctica para instalar, configurar, desplegar y entender DevFast Manager.

Instalación

git clone https://github.com/UserZero075/Business-Management-Dashboard.git
cd Business-Management-Dashboard/backend
cp .env.example .env  # si existe; si no, crea backend/.env
npm install
npx prisma generate
npx prisma db push
npm run dev

cd ../frontend
npm install
VITE_API_URL=http://localhost:3001 npm run dev

Variables principales

DATABASE_URL

SQLite: file:./devfast.db. Puede migrarse a MySQL/PostgreSQL cambiando Prisma.

JWT_SECRET

Secreto fuerte para firmar sesiones.

SMTP_*

Servidor de correo para OTP de registro.

VITE_API_URL

URL pública del backend usada por el frontend.

Despliegue

En VPS puedes usar deploy.sh: sube el proyecto o un tar.gz, ejecuta chmod +x deploy.sh && ./deploy.sh. Revisa puertos, firewall, variables de entorno y proxy inverso antes de producción.

API

El backend expone rutas REST bajo /api para auth, projects, vps, finance, tasks, dashboard, team, chat y reports. El README del repo lista los endpoints principales.

Arquitectura

Frontend React/Vite consume una API Fastify TypeScript. Prisma gestiona el modelo de datos. WebSocket se usa para chat con fallback HTTP.
